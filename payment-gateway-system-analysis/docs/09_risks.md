# Риски и ограничения / Risks and Limitations

## Риски проекта / Project Risks

| Риск | Описание | Как снизить риск |
|---|---|---|
| Дублирование платежей | Повторный запрос от мерчанта может создать второй платеж по одному заказу | Использовать `Idempotency-Key` и уникальные ограничения в базе данных |
| Потеря результата оплаты | Callback от банка-эквайера может не быть обработан из-за ошибки системы | Сохранять попытки оплаты и технические логи |
| Некорректный статус платежа | Платеж может перейти в недопустимый статус | Использовать строго заданную state machine |
| Ошибка доставки webhook | Система мерчанта может быть недоступна в момент отправки уведомления | Сохранять webhook-событие и предусмотреть повторную отправку |
| Несогласованность данных | Статус платежа в платежном шлюзе и у мерчанта может различаться | Использовать webhook и endpoint получения статуса платежа |
| Ошибки интеграции | Мерчант может отправить некорректный запрос | Валидировать входные данные и возвращать понятные ошибки API |

## Ограничения MVP / MVP Limitations

| Ограничение | Описание |
|---|---|
| Нет реального банка-эквайера | В проекте используется заглушка банка-эквайера |
| Нет обработки банковских карт | Система не принимает, не хранит и не передает данные банковских карт |
| Нет PCI DSS | Соответствие PCI DSS не входит в рамки MVP |
| Нет возвратов | Refund-сценарии будут добавлены на следующем этапе |
| Нет холдирования средств | Статусы AUTHORIZED и CAPTURED не используются в MVP |
| Нет личного кабинета мерчанта | Управление мерчантом не реализуется в текущей версии |
| Нет антифрода | Проверка подозрительных операций не входит в рамки MVP |

## Дальнейшее развитие / Future Improvements

| Направление | Описание |
|---|---|
| Refunds | Добавить полные и частичные возвраты |
| Payment hold | Добавить авторизацию и последующее списание средств |
